## 2. test-results v1.2.0 — per-invariant notes

- **B-RES-001 (storage disposition)** — zen: after an `admit` verdict and
  BEFORE idempotency/storage, `POST /api/results` with `mode=zen` returns
  HTTP 200 `{ verdict: "admit", stored: false, anticheat: { decision, reasons } }`.
  No record is written; repeated zen submissions never create records; history
  never contains `mode=zen`. The recorded anticheat decision is echoed so the
  RVL monitorable projection on this route (`body.anticheat.decision === "admit"`
  for 2xx POSTs) keeps holding — verified by unit test. Rejected zen still
  422s (never stored either). Response shape is the delegated cosmetic the
  ambiguity-log reserves (§6.1).
- **B-RES-001 (C6)** — `minThresholdFailed === true` IS persisted on the
  record (absent ⇒ `false`), visible in history.
- **B-RES-002/003/004/005** — idempotency, strict-improvement PB transitions,
  bailed exclusion, newest-first history unchanged. v1.2.0 exclusions: the PB
  gate now reads `!stored.bailedOut && !stored.minThresholdFailed` — flagged
  results never become PB (`isPb` stays false) and never demote the standing
  PB (tested: flagged 200 after PB 100 ⇒ PB stays 100; unflagged 150 then
  takes PB — the flagged 200 never counted as best). PB reads filter `isPb`,
  so flagged results are excluded from `/api/results/pbs` by construction.
- **B-RES-006 (tags composite)** — store `data/tags.json` `{ tags: [{id, uid, name}] }`
  (pre-authorized capability); wire shape exactly `{id,name}` (ajv-checked vs
  `tag.schema.json`). (a) CRUD: create 201 / rename PATCH / delete DELETE;
  names unique per user **case-insensitively** (409 `conflict` on create +
  rename clash; rename to own current name OK); name domain 1..64 (422).
  (b) assignment: `POST /api/results/:id/tags {tagId}` / `DELETE
  /api/results/:id/tags/:tagId`; own results + own tags only — unknown or
  foreign yields an indistinguishable 404 `not_found` envelope; assign
  idempotent (no duplicate entries); ≤20 tags per result (stored-result
  schema `maxItems`). (c) filter: `GET /api/results?tags=id1,id2` —
  **multi-tag = intersection** (sealed). (d) delete-cascade: tag deletion
  removes the id from every own result (results untouched otherwise); two
  logical writes, inside the `max_writes_per_request: 2` capability.
  (e) tag-scoped PB: `GET /api/results/pbs?tags=...` — pure read-time
  derivation (`scopedPbs`): best per sealed C7 tuple among own results
  carrying ALL filter tags, excluding bailed + minThresholdFailed (same
  exclusions as global PB reads); **never creates/mutates/demotes isPb**
  (test asserts flags byte-identical before/after scoped reads). Global C7
  keying per B-RES-003 stands.
- **S-RES-001/002/003** — intake validator extended for the v2 optional
  fields; stored records ajv-conform to the amended stored-result schema
  (incl. `minThresholdFailed`, `tags`) — unit-tested; all failures ErrorEnvelope.

- **B-STS-002 (recompute-consistency)** — formulas exactly as sealed, in
  `shared/resultStats.js`: per-(mode, mode2) `testsCompleted` = count,
  `timeTypingSeconds` = **sum(testDuration)** (round-5 ruling BQ-STS-01:
  afkDuration NOT subtracted), `avgWpm/avgAcc` = means; PB table = exactly one

## 5. quote-library v1.1.0 — per-invariant notes

- **B-QT-006 (tri-state moderation)** — every new quote persists
  `state ∈ {pending, approved, refused}` + `approved` boolean with the (a)
  consistency clause maintained on every mutation; legacy stored quotes
  (incl. pre-v1.1 data and old KV) derive `state` from the boolean at serve
  time (`quoteState`) — zero migration, defaults-merge style. Serve shape
  carries `state` + optional `moderationNote` (≤500, 422 beyond);
  `serveQuote` emits exactly the sealed quote schema fields
  (additionalProperties:false; ajv-checked). (b) `POST /api/quotes/:id/approve`
  and `/refuse` are moderator-only (403) and idempotent, optional
  `moderationNote` persisted. (c) refused quotes are PERSISTED with their
  metadata (direct moderation reads still return them; favoriting a refused
  id works) and NEVER served — random/search/browse/favorites-list filter
  `state === "approved"` (tested over 12 random draws + substring probes).
  (d) pending never served either (B-QT-001 stands).
- **B-QT-007 (rating weight)** — documented weight function
  `weight(q) = rating average` for rated quotes, **unrated default = 2.5**
  (neutral midpoint of 1..5) — monotonically non-decreasing in the average by
  construction (module test sweeps averages). Random fetch is a weighted pick
  (`weightedPickIndex`, cumulative-weight mapping); optional `?seed=<uint>`
  selects via `mulberry32(seed)` ⇒ same pool + weights + seed = same quote
  (module + HTTP tested); unseeded reads keep `Math.random`. Monotonicity is
  sealed on the function, not on sampled outcomes — no flake budget consumed.
- **B-QT-008 (favorites)** — `data/favorites.json` `{ favorites: { uid: [quoteId…] } }`
  (pre-authorized capability). `POST /api/quotes/favorites` validates the
  sealed favorite-request schema strictly (extra keys ⇒ 422); add idempotent;
  unknown quote 404. `GET` returns only APPROVED quotes the user favorited
  (add-order stable). `DELETE /api/quotes/favorites/:quoteId` idempotent and
  never deletes the quote (post-remove browse probe still finds it).
  One store write per mutating request (O-QT-001).
- **B-QT-009 (search/browse)** — `GET /api/quotes?language=&q=&page=`:
  approved-only; `language` exact, `q` case-insensitive text substring;
  **stable order = submission (storage) order**; **page size fixed 50**,
  `page` 0-based (invalid ⇒ 422). Response `{ quotes, page, pageSize, total }`.
  Module sweep over 120 synthetic quotes proves pagination is total
  (every quote exactly once, in order).
- **B-QT-002/003/004/005, S-QT-001/002** — unchanged mechanics (groups,
  idempotent dedupe resubmission returns the existing quote in ANY state with
  no second write, moderator gate, replace-on-re-rate); harness regression green.

## 6. leaderboards v1.1.0 — per-invariant notes

- **S-LB-001/002** — board key `(mode=time, mode2 ∈ {15,60}, language,
  timeWindow ∈ {alltime, daily})`; `?language=` (default `english`) must name
  a wordlists registry entry — validated against the boot-admitted registry
  (`REGISTRY_IDS`); unknown language / bad mode2 / bad window ⇒ 404
  ErrorEnvelope. Responses ajv-conform to the sealed leaderboard schema
  (this is the check that turned the loop from red to green — §8).
- **B-LB-001** — eligibility = anticheat admit ∧ `!bailedOut` ∧
  `minThresholdFailed !== true` (C6 chain lands consumer-side, as recorded).
  One entry per user = their best ELIGIBLE result (tested: flagged 200 +
  bailed 300 invisible, best eligible 100 wins).
- **B-LB-002/003/004** — ordering (wpm desc, earlier timestamp tie-break),
  read-time recomputation (no cache — a fresh result is visible on the next
  read, tested), requester null when anonymous / no eligible entry, otherwise
  rank + entry even outside top-N.
- **B-LB-005 (daily)** — rolling **(T-24h, T]** at read time (`Date.now()`;
  `inWindow` is pure under an injected clock — exact open-left/closed-right
  edges module-tested; 25h-old excluded, 23h-old included over HTTP).
  `alltime` = full history. BQ-LB-02: same top-N as alltime (N≤100, default 50).
- **B-LB-006 (percentile)** — every entry AND the requester field carry
  `percentile = 100 · rank / totalEligibleUsers` (round2 for stable floats;
  sealed example 1/200 ⇒ 0.5 module-tested; 3-user board ⇒ 33.33/66.67/100
  HTTP-tested). `totalEligibleUsers` = distinct users with an eligible entry
  on that board at read time (= entries before slicing).
- **B-LB-007 (XP)** — read-time per-entry derivation, zero writes.
  Documented formula (delegated coefficients, §6.9):
  `xp = wpm · (acc/100) · (testDuration/60)` — one XP per adjusted word
  (gross words discounted by accuracy). Sealed inputs only (wpm, acc,
  testDuration); deterministic (same result ⇒ same xp, tested); monotonic
  non-decreasing in each input. Aggregate XP/levels stay user-profile (D8).

## 7. Delegated decisions (CA-001 — decided + logged)

1. **Zen non-stored response shape** (ambiguity-log reserves it as cosmetic):
   HTTP 200 `{ verdict, stored: false, anticheat: { decision, reasons } }` —
   200 not 201 (nothing created); `anticheat` echo keeps the RVL monitorable
   projection valid on this route.
2. **Wordlists as real static JSON files** (not a bundled JS module —
   contrast with theme-catalog §5.1): S-WL-003 seals "served as same-origin
   static files"; the capability read-list names `assets/wordlists/`.
   express.static serves bytes verbatim; the Workers build embeds the same
   bytes. Single data authority: the JSON files (node reads at boot; worker
   embeds at build).
3. **Retired internal provider kept as export** (`internalWordlist`,
   `ENGLISH_200`): (a) the sealed engine v2 harness checks import it — the
   formal validator migration is the later stage, and breaking the sealed
   validator surface is out of scope; (b) client offline/degraded fallback
   only. Runtime word source for shipped sessions is the wordlists asset via
   the handshake. `english.json ≡ ENGLISH_200` parity-tested so the two
   authorities cannot silently drift.
4. **Route names (unsealed surfaces)**: stats `/api/stats/{aggregates,pbs,
   activity,wpm-series}`; tags `/api/results/tags[...]` + assignment
   `/api/results/:id/tags[...]`; favorites `/api/quotes/favorites[...]`;
   search `GET /api/quotes`; leaderboard params `?language=&timeWindow=&n=`;
   wordlists `/wordlists/registry.json`, `/wordlists/<id>.json`.
5. **Invalid leaderboard key components ⇒ 404 `not_found`** ("no such board"
   semantics), extending the sealed v1 mode2 behavior to language/timeWindow.
6. **Tag-filter query accepts any ids** (unknown ids simply match nothing —
   cascade guarantees no result can carry one); assignment is the fail-closed
   surface (404), per the sealed (b)/(c) split.
7. **Scoped PB tie-break** = earlier timestamp (mirrors LB tie-break; global
   PB maintenance never faces ties due to strict improvement).
8. **Quote resubmission dedupe returns the existing quote in ANY state**
   (sealed: "returns the existing quote without a second write") — a refused
   text stays refused (documented; re-opening would be a behavior change for
   a later negotiation).
9. **XP coefficients**: `xp = wpm · (acc/100) · (testDuration/60)`, rounded
   to 2 decimals — the simplest dimensionally-sound form over the sealed
   inputs (adjusted correct words); coefficients documented here per
   B-LB-007's delegation.
10. **Rating weight**: `weight = average` (rated), `2.5` unrated default
    (documented midpoint); `?seed=` opt-in reproducibility keeps unseeded
    behavior identical to v1.
11. **Search order = submission order, pageSize 50 fixed, page 0-based** —
    deterministic and total (pagination proof in tests).
12. **Starter lists**: 6 single-tier lists, ids == language names (the
    list-level-id design subsumes `english_1k`-style variants when the data
    task lands); `tier` metadata carried for forward compat.
